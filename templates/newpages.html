New Pages SPARQL endpoint

Data Loading

Use one or more comma-separated Wikimedia wiki identifiers such as fi, se, smn, olo, b:fi, commons, wikidata, incubator, or meta. Bare language codes default to Wikipedia, so fi means fi.wikipedia.org. Full hostnames such as fi.wikipedia.org are still accepted and normalized automatically. Supported projects: Wikipedia, Wiktionary, Wikibooks, Wikinews, Wikiquote, Wikisource, Wikiversity, Wikivoyage, Wikidata, Commons, Incubator, and Meta-Wiki. Wildcards such as *.wikipedia.org are also supported.

Accepts YYYYMMDDHHMMSS prefixes. Shorter values are right-padded with zeros, so YYYYMM becomes YYYYMM00000000.

Optional. Use a wiki page that links to user pages. You can paste either an interwiki page reference such as :w:fi:Wikipedia:Viikon_kilpailu/Viikon_kilpailu_2026-15 or a direct Wikimedia page URL; the UI normalizes direct links to interwiki format automatically.

Optional. When enabled, the timestamp filter is matched against revision timestamps instead of page creation time, and it must stay within the last 60 days. Matching rows then expose current_timestamp as the latest matching revision time for each page.

Default is 10. Clear this field to use the backend cap instead of a smaller manual limit: API mode scans up to 100 newest creation-log entries per wiki, and SQL mode is capped at 50000 rows.

About New Pages

This source reads recentchanges rows where rc_source="mw.new" from one or more Wikimedia wikis validated against SiteMatrix. The wiki filter accepts short identifiers such as fi, b:fi, commons, and wikidata. With User list page plus the edited-pages option, SQL mode switches to revision-based matching for the last 60 days. The result graph uses schema.org sitelink-style triples, so the returned pages can be joined into federated Wikidata queries in the same way as the Incubator source.
